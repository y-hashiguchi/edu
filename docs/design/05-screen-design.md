## 2. 画面遷移

```
                        ┌────────┐
        起動 ───────▶  │ /login │
                        └───┬────┘
                            │ ログイン成功
                            ▼
                        ┌────────┐
            ◀───────── │   /    │ ◀──── 「一覧に戻る」
            (ロック中) └───┬────┘
                            │ ロック解除フェーズの「対話する」
                            ▼
                        ┌───────────────┐
                        │ /phases/:phase│
                        └───┬───────────┘
                            │ 「このフェーズを完了する」
                            ▼
                        progress 完了 → 次フェーズ解放
                            │
                            └─▶ / に戻る
```

- 未認証で `/` または `/phases/:phase` を直接開いた場合 → `/login` にリダイレクト
- ログイン済で `/login` を開いた場合 → `/` にリダイレクト
- 401 受信時 → トークン破棄 → `/login` にリダイレクト

## 3. S-LOGIN: ログイン / 新規登録

### 3.1 ワイヤーフレーム

```
┌─────────────────────────────────────────────┐
│  AI駆動型開発 補足カリキュラム                  │
│  AIチューター — 学習サポートシステム             │
├─────────────────────────────────────────────┤
│                                                 │
│   ┌──────────────────────────────┐ │
│   │  [ ログイン ]  [ 新規登録 ]            │ │ ← タブ切替
│   ├──────────────────────────────┤ │
│   │                                          │ │
│   │  メールアドレス                            │ │
│   │  [______________________]                │ │
│   │                                          │ │
│   │  パスワード                                │ │
│   │  [______________________]                │ │
│   │                                          │ │
│   │  （新規登録時のみ）                          │ │
│   │  お名前                                    │ │
│   │  [______________________]                │ │
│   │                                          │ │
│   │   ⚠ メールアドレスまたはパスワードが        │ │ ← エラー時
│   │      正しくありません                       │ │
│   │                                          │ │
│   │      [   ログイン   ]                      │ │
│   │                                          │ │
│   └──────────────────────────────┘ │
│                                                 │
└─────────────────────────────────────────────┘
```

### 3.2 構成要素

| 要素 | 種別 | 必須 | バリデーション |
|---|---|---|---|
| タブ切替 | ボタン2つ | — | ログイン / 新規登録 |
| email | input[type=email] | ○ | RFC、255 文字以内 |
| password | input[type=password] | ○ | 8〜128 文字 |
| name | input[type=text] | ○（登録時） | 1〜100 文字 |
| 送信ボタン | button[type=submit] | — | 送信中は disabled |
| エラーバナー | div[role=alert] | — | API エラー表示 |

### 3.3 状態と振る舞い

| 状態 | 表示 |
|---|---|
| 初期 | フォーム + ボタン活性 |
| 送信中 | ボタン disabled、ローダー表示 |
| 失敗 | エラーバナー表示、ボタン再活性化 |
| 成功（ログイン） | `/` へ遷移 |
| 成功（登録） | ログインタブに自動切替、「登録できました。ログインしてください」を緑バナー表示 |

### 3.4 エラー表記

| HTTP | 表示テキスト |
|---|---|
| 401 | 「メールアドレスまたはパスワードが正しくありません」 |
| 409（登録時） | 「このメールアドレスは既に登録されています」 |
| 422 | 「入力内容を確認してください」（各フィールド下にも詳細表示） |
| その他 / ネットワーク | 「通信に失敗しました。時間をおいて再試行してください」 |

### 3.5 イベント

| トリガ | アクション |
|---|---|
| タブクリック | `mode = 'login' \| 'register'` 切替 |
| Enter キー（フォーム内） | 送信 |
| 送信ボタン | `authStore.login()` or `authStore.register()` |
| 成功 | router.push('/') または mode 切替 |

## 4. S-HOME: フェーズ一覧

### 4.1 ワイヤーフレーム

```
┌─────────────────────────────────────────────┐
│  AI駆動型開発 補足カリキュラム  | アリスさん | ログアウト │
│  AIチューター — 学習サポートシステム             │
├─────────────────────────────────────────────┤
│                                                 │
│  あなたの進捗 ●○○○ （1/4 完了）                  │
│                                                 │
│  ┌────────────┐  ┌────────────┐ │
│  │ PHASE 1 [進行中] │  │ PHASE 2 [ロック]  │ │
│  │ 開発環境の近代化  │  │ AIツール活用    │ │
│  │ 2〜3週間         │  │ 3〜4週間         │ │
│  │                  │  │                  │ │
│  │ 学習スキル        │  │ 🔒              │ │
│  │ ・Git / GitHub   │  │ Phase 1 を完了   │ │
│  │ ・VSCode拡張     │  │ すると解放されます│ │
│  │ ・ターミナル操作  │  │                  │ │
│  │ ・REST API基礎   │  │                  │ │
│  │                  │  │                  │ │
│  │ 課題              │  │                  │ │
│  │ 1. Gitでブランチ… │  │                  │ │
│  │ 2. VSCode拡張…   │  │                  │ │
│  │ 3. curlでAPI…   │  │                  │ │
│  │                  │  │                  │ │
│  │ [AIチューターと対話 →] │  │                  │ │
│  └────────────┘  └────────────┘ │
│                                                 │
│  ┌────────────┐  ┌────────────┐ │
│  │ PHASE 3 [ロック]  │  │ PHASE 4 [ロック]  │ │
│  │ ...              │  │ ...              │ │
│  └────────────┘  └────────────┘ │
│                                                 │
└─────────────────────────────────────────────┘
```

### 4.2 構成要素

| 要素 | 種別 | 用途 |
|---|---|---|
| ヘッダー | 共通 | ユーザ名 + ログアウト |
| 進捗バー | 進捗の集計 | `completed` 数 / 4 |
| PhaseCard | 4 枚 | フェーズ詳細 + CTA |

### 4.3 PhaseCard 状態

| status | バッジ | CTA | 副作用 |
|---|---|---|---|
| `in_progress` | 「進行中」（青） | 「AIチューターと対話する →」 → `/phases/n` | — |
| `submitted` | 「提出済み」（黄） | 「AIチューターと対話する →」 → `/phases/n` | Sprint 1 では未使用 |
| `completed` | 「完了」（緑） | 「AIチューターと対話する →」 → `/phases/n` | 完了済でも対話可能 |
| `locked` | 「ロック」（灰） | 押下不可（カード全体グレースケール）、ロック理由文 | aria-disabled |

ロック理由：
- Phase 2: 「Phase 1 を完了すると解放されます」
- Phase 3: 「Phase 2 を完了すると解放されます」
- Phase 4: 「Phase 3 を完了すると解放されます」

### 4.4 ローディング / エラー

| 状態 | 表示 |
|---|---|
| ロード中 | 全画面 spinner + 「読み込み中…」 |
| API エラー | カードグリッドの代わりに赤バナー + 「再読み込み」ボタン |
| データ空 | （想定外）「データを取得できませんでした」 |

### 4.5 イベント

| トリガ | アクション |
|---|---|
| onMounted | `curriculumStore.fetchPhasesWithProgress()`（phases + progress を並列取得） |
| CTA クリック（lock=false） | `router.push('/phases/n')` |
| CTA クリック（lock=true） | 何もしない（disabled） |
| ログアウト | `authStore.logout()` → `router.push('/login')` |

## 5. S-CHAT: フェーズ詳細 + AI チューターチャット

### 5.1 ワイヤーフレーム

```
┌─────────────────────────────────────────────┐
│  ヘッダー（共通）                                 │
├─────────────────────────────────────────────┤
│  ← 一覧に戻る                                    │
│                                                 │
│  Phase 1 — 開発環境の近代化                        │
│  AIツールを使いこなすための「土台」を固める            │
│                                                 │
│  ┌──── クイック質問 ────────────────┐ │
│  │  [Gitでブランチを切り、…           ]  │ │
│  │  [VSCode拡張（GitLens…）          ]  │ │
│  │  [curlでREST APIを叩き…           ]  │ │
│  └──────────────────────────────┘ │
│                                                 │
│  ┌──── 会話ログ（スクロール） ─────────────┐ │
│  │                                            │ │
│  │  あなた: Gitのブランチって何？               │ │
│  │  チューター: いい質問ですね。Gitのブランチは… │ │
│  │  あなた: コミットとの違いは？                │ │
│  │  チューター: ...                          │ │
│  │                                            │ │
│  └──────────────────────────────┘ │
│                                                 │
│  ┌──────────────────────────────┐ │
│  │ メッセージを入力                          │ │
│  │ [_________________________________]      │ │
│  │                              [ 送信 ]    │ │
│  └──────────────────────────────┘ │
│                                                 │
│  ─────────────────────────────────────────── │
│  [ このフェーズを完了する ]                       │
│                                                 │
└─────────────────────────────────────────────┘
```

### 5.2 構成要素

| 要素 | 種別 | 用途 |
|---|---|---|
| 戻るリンク | router-link | `/` へ |
| ヘッダー | h1 + 説明 | フェーズタイトル / goal |
| QuickQuestion | ボタン × 3 | クリックで自動メッセージ送信 |
| ChatLog | スクロールエリア | role により左右配置 |
| ChatInput | input + ボタン | メッセージ送信 |
| 完了ボタン | button | 「このフェーズを完了する」 |

### 5.3 状態

| 状態 | 表示 |
|---|---|
| 初期ロード | 「フェーズ情報を読み込み中…」 |
| 履歴ロード成功 | ChatLog にメッセージ列描画 |
| 送信中 | ChatInput / QuickQuestion を disabled、最下部に「AIが応答中…」 |
| 送信失敗 | 赤バナー「エラーが発生しました。再試行してください」、入力欄活性化 |
| ロックフェーズへの直接遷移 | onMounted で `/` にリダイレクト |
| 完了済フェーズ | 完了ボタンは押下可能（再完了は冪等） |
| 完了処理中 | 完了ボタン disabled、確認ダイアログ |

### 5.4 ChatMessage の表示規則

| role | 配置 | 背景色 | アイコン |
|---|---|---|---|
| user | 右寄せ | 青系 | 👤 |
| assistant | 左寄せ | 白 / グレー枠 | 🤖 |

長文は折り返し、コードブロックは monospace で表示（Sprint 1 では markdown レンダリングは未実装、Sprint 2+ で `marked` 等の導入を検討）。

### 5.5 完了確認ダイアログ

```
┌──────────────────────────────────┐
│  Phase 1 を完了しますか？               │
│                                       │
│  完了すると Phase 2 が解放されます。     │
│  履歴は引き続き閲覧できます。            │
│                                       │
│           [ キャンセル ] [ 完了する ]    │
└──────────────────────────────────┘
```

最終フェーズ（Phase 4）完了時はメッセージを「Phase 4 を完了しますか？すべてのカリキュラムが終わります。」に変更。

### 5.6 イベント

| トリガ | アクション |
|---|---|
| onMounted | `fetchPhasesWithProgress()`（未取得時）→ `loadHistory(phase)` → ロック時 `/` 遷移 |
| QuickQuestion クリック | `submit(\`課題について教えてください: ${task}\`)` |
| 送信ボタン | `submit(inputText)` → `inputText = ''` |
| Enter（Shift以外） | 送信 |
| 完了ボタン | 確認ダイアログ表示 → 「完了する」で `curriculumStore.completePhase(phase)` → `/` へ遷移 |

## 7. アクセシビリティ

| 項目 | 対応 |
|---|---|
| 入力ラベル | `<label for>` で必ず紐付け |
| エラー表示 | `<div role="alert" aria-live="polite">` |
| ロック CTA | `aria-disabled="true"` + 視覚的グレースケール |
| フォーカス順 | DOM 順で自然に。`tabindex` は使わない |
| キーボード操作 | Enter で送信、Shift+Enter で改行（ChatInput） |
| カラーコントラスト | テキスト/背景は WCAG AA 4.5:1 を満たす |
| 動的更新 | `aria-live` でメッセージ追加を読み上げ |
