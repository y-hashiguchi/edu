## スコープと前提

本設計書群は **Sprint 1（DB + 認証 + 進捗管理 + 会話履歴永続化）** をカバーする。Sprint 0（スケルトン + AI チューター対話 MVP）の完了状態を起点とする。

| Sprint | スコープ | 状態 |
|---|---|---|
| 0 | スケルトン、4フェーズデータ、`/api/curriculum/phases`、`/api/chat`（メモリ）、Vue 2 画面、Docker Compose | **完了** |
| **1** | **Postgres + Alembic、JWT 認証、進捗管理（解放ロック）、会話履歴 DB 永続化、フロントの認証 UI / ロック表示 / 履歴ロード** | **本書対象** |
| 2 | 課題提出（テキスト/ファイル）+ AI 採点、pgvector 利用、ローカル LLM 評価 | 後続 |
| 3 | 管理者ダッシュボード（全受講者進捗・コメント・通知） | 後続 |
