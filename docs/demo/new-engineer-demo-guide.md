# 新人エンジニア向け デモ操作マニュアル

AI チューター学習プラットフォーム（edu）の公開デモを、はじめての方が一通り体験するためのガイドです。アカウント登録から、AI への質問、課題提出と AI 採点、フェーズ完了までを順番に操作します。

- **所要時間:** 約 10〜15 分
- **対象:** 新人エンジニア（このシステムを初めて触る人）
- **必要なもの:** Web ブラウザ（Chrome / Firefox / Safari の最新版推奨）とメールアドレス形式の文字列（実在しなくても可。例: `taro@example.com`）

> このデモは公開パイロット環境です。学習用のサンプルであり、入力したデータは他の体験者から見えない自分専用の領域に保存されます。

---

## 0. アクセス先

| 用途 | URL |
|---|---|
| デモ画面（これを開く） | <https://edu-demo-web.onrender.com> |
| （参考）API ヘルスチェック | <https://edu-demo-api.onrender.com/healthz> |

> ⏳ **初回アクセスが遅いことがあります。** デモはコスト最小構成で動いており、しばらく無アクセスだとサーバーがスリープします。最初の表示や最初の応答に **30〜60 秒**かかる場合がありますが、2 回目以降は速くなります。エラーが出たら一度時間をおいて再読み込みしてください。

---

## 1. アカウントを登録する

1. ブラウザで <https://edu-demo-web.onrender.com> を開きます。未ログインなのでログイン画面が表示されます。
2. 画面上部のタブで **「新規登録」** を選びます。
3. フォームに入力します。
   - **メールアドレス**: 任意（例 `taro@example.com`）。実在しなくてもOKですが、後でログインに使うので覚えておいてください。
   - **受講コース**: プルダウンから選択（必須）。初めてなら **「AI駆動型開発 補足カリキュラム」** がおすすめです。
   - **パスワード**: **8 文字以上**（例 `Passw0rd!`）。
4. **「登録する」** をクリックします。
5. 「**登録できました。続けてログインしてください。**」と表示されれば成功です。

> 💡 コースは登録時に1つ選びます。選んだコースに自動的に受講登録（enrollment）され、最初のフェーズ（Phase 1）が解放された状態で始まります。

### 選べるコース

| コース | 内容 |
|---|---|
| AI駆動型開発 補足カリキュラム (`ai-driven-dev`) | 既存 Java/Python 経験者向け。4 フェーズ。**デモ体験はこちらが手軽** |
| AI時代SE育成カリキュラム (`ai-era-se`) | 12 ヶ月の SE 育成（4 フェーズ・31 課題）の本格カリキュラム |

---

## 2. ログインする

1. 画面上部のタブが **「ログイン」** になっていることを確認します。
2. 登録した **メールアドレス** と **パスワード** を入力します。
3. **「ログイン」** をクリックします。
4. ログインに成功すると、受講コースの画面（コースが1つなら自動でそのコースのホーム）へ移動します。

> ❗ うまくいかないとき: 「メールアドレスまたはパスワードが正しくありません」と出たら、登録時の値と一致しているか確認してください（パスワードは8文字以上）。

---

## 3. コースとフェーズを開く

1. ログイン後、**コースのホーム画面**にフェーズの一覧（Phase 1〜4）が表示されます。
2. 各フェーズはカードで表示され、**解放済み（Phase 1）** と **ロック中（Phase 2 以降）** が区別されます。
3. **Phase 1「開発環境の近代化」** を開きます。

> 🔒 Phase 2 以降は、前のフェーズを「完了」すると順番に解放されます（後述のステップ6）。

Phase 1 の課題は次の3つです（このマニュアルでは課題1を使います）。

| 課題 | タイトル |
|---|---|
| 課題1 | Gitでブランチを切り、PythonスクリプトをプッシュしてPRを作成 |
| 課題2 | VSCode拡張（GitLens・REST Client・GitHub Copilot）の導入と動作確認 |
| 課題3 | curlでREST APIを叩き、JSONレスポンス構造をまとめる |

---

## 4. AI チューターに質問する

Phase 画面では、AI チューターにチャットで質問できます。

### 4-1. クイック質問（ワンクリック）

1. **「クイック質問」** の下にあるボタンをクリックします。
2. 定型の質問が AI に送信され、しばらくすると回答がチャット欄に表示されます。

### 4-2. 自由入力で質問

1. 画面下部のチャット入力欄（プレースホルダ「**質問を入力（Cmd/Ctrl+Enter で送信）**」）に質問を入力します。
   - 例: 「Gitのブランチ運用について、新人向けに3点で教えてください。」
2. **「送信」** ボタン、または **Cmd/Ctrl + Enter** で送信します。
3. 数秒〜十数秒で AI チューターの回答が表示されます。
4. 会話履歴は保存されます。ページを再読み込みしても、同じフェーズのやり取りは残っています。

> ⏳ AI の回答には数秒〜十数秒かかります。送信直後はボタンが無効化され「送信中」の状態になります。

---

## 5. 課題を提出して AI 採点を受ける

1. Phase 画面の **「課題提出」** セクションを開きます（課題ごとに提出カードがあります）。
2. **提出内容**欄（プレースホルダ「**提出内容を記入してください...**」）に、課題への取り組み内容を記入します。
   - 例（課題1）: 「`feature/hello` ブランチを作成し、`hello.py` をコミットして origin にプッシュ、main への PR を作成しました。PR URL: …」
3. 必要に応じて**ファイルを添付**します（任意。対応拡張子: py, java, js, ts, txt, md, png, jpg, jpeg, pdf／1課題あたり最大3ファイル・1ファイル5MBまで）。
4. **「提出する」** をクリックします。
5. AI による採点が走り、しばらくすると以下が表示されます。
   - **スコア**（100点満点）
   - **AI フィードバック**（良かった点と改善点）
6. 内容を修正して **「再提出する」**、または **「再採点」** で採点をやり直せます（短時間に連続した再採点はクールダウンで制限されます）。

> 💡 採点は AI（Claude）が行います。提出内容が具体的なほど、フィードバックも具体的になります。

---

## 6. フェーズを完了して次へ進む

1. フェーズの課題に取り組んだら、画面下部の **「このフェーズを完了する」** ボタンをクリックします。
2. 確認ダイアログ「**Phase 1 を完了しますか？**」「**完了すると Phase 2 が解放されます。履歴は引き続き閲覧できます。**」が出ます。
3. **「完了する」** をクリックします。
4. 完了すると **Phase 2** が解放され、コースホームから開けるようになります。完了済みフェーズのチャット・提出履歴は引き続き閲覧できます。

これで受講者の基本フロー（登録 → ログイン → 質問 → 提出 → 採点 → フェーズ完了）は一通り体験完了です。お疲れさまでした。

## トラブルシューティング

| 症状 | 原因・対処 |
|---|---|
| 最初の画面表示や応答がとても遅い | デモのサーバーがスリープからの復帰中。30〜60 秒待って再読み込み |
| ログインできない | メール/パスワードの打ち間違い。パスワードは8文字以上 |
| 「このメールアドレスは既に登録されています」 | 別のメールアドレスで登録するか、そのアドレスでログイン |
| AI の回答が返ってこない / エラー | 数秒待って再送信。続く場合は時間をおいて再試行 |
| 課題提出でファイルが弾かれる | 対応拡張子・サイズ・ファイル数の上限を確認（py/java/js/ts/txt/md/png/jpg/jpeg/pdf、5MB、3ファイルまで） |
| Phase 2 以降が開けない | 前のフェーズを「完了する」と解放されます |

## 付録 B: 開発者向け — API で動作確認する方法

ブラウザを使わず、コマンドラインから同じ流れを確認できます（学習・検証用）。

```bash
API=https://edu-demo-api.onrender.com
COURSE=ai-driven-dev
EMAIL="you-$(date +%s)@example.com"
PASS="DemoTest-pw-12345"

# 1) 登録
curl -s -X POST "$API/api/auth/register" -H "Content-Type: application/json" \
  -d "{\"email\":\"$EMAIL\",\"name\":\"Demo\",\"password\":\"$PASS\",\"course_slug\":\"$COURSE\"}"

# 2) ログイン → トークン取得
TOKEN=$(curl -s -X POST "$API/api/auth/login" -H "Content-Type: application/json" \
  -d "{\"email\":\"$EMAIL\",\"password\":\"$PASS\"}" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 3) AI チャット
curl -s -X POST "$API/api/chat?course=$COURSE" -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"phase":1,"message":"Gitのブランチ運用を新人向けに3点で。"}'

# 4) 課題提出（同期採点なのでレスポンスに score / ai_feedback が入る）
curl -s -X POST "$API/api/submissions?course=$COURSE" \
  -H "Authorization: Bearer $TOKEN" \
  -F "phase=1" -F "task_no=1" \
  -F "content=feature/hello ブランチを作成しPRを作成しました。"
```

> ⚠️ 体験で作成したアカウントはデモ DB に残ります。検証用に作ったアカウントは運用上無害ですが、本番運用に切り替える際は整理してください。
